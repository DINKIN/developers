API Endpoints

The API expoeses resources through various HTTP endpoints. You work with these resources through consistent use of HTTP verbs and a consistent tree of endpoints. The schema for the various resources available is documented in the objects section.

Lists and objects

There are generally two kinds of resources you can retrieve with a GET request - lists and objects. A list is a collection of objects, and each object has an ID and may be retrieved individually. You'll generally find lists of objects at /object-name, and the individual objects at /object-name/object-id. Some objects contain lists of other objects - you can get these at /object-name/object-id/subobject-name, and you can get an individual one at /object-name/object-id/subobject-name/subobject-id. For example, you can list your Linodes at /linodes, and get a specific Linode /linodes/lino_1234. You can get the configs for this Linode from /linodes/lino_1234/configs.

Pagination

Resource lists are always paginated. The response looks something like this: 

{
    "linodes": [ ... ],
    "page": 1,
    "total_pages": 10,
    "total_results": 248
}

Pages start at 1. Get a particular page by adding ?page=2 to your URL. Each page has 25 results.

Editing objects

Some objects are editable, and each editable object may only have a handful of properties you may edit. the properties you may edit are indicated in the objects section, with this indicator: Editable. To edit these objects, you can perform a PUT request against the resource whose body is the updated JSON. You can omit any fields, or include fields you are not permitted to edit (changes to the latter will be ignored). In other words, if you GET a resource, you can change a property in the JSON that comes back and then PUT that JSON to the same URL to update it.

HTTP Methods

GET
Gets information about a resource or resources
PUT
Edits information about a resource
POST
Creates a new resource
DELETE
Deletes a resource
